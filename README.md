# 🇰🇷 프로젝트 소개

Baseball Match Tracker는 KBO 리그가 2030 여성층까지 팬층을 넓히며 1,000만 관객을 돌파하던 시기에, 급성장하는 야구 시장 속에서 사용자들이 어떤 상호작용형 서비스를 필요로 할지 고민하며 기획한 프로젝트입니다.  
경기 정보를 조회하는 것 뿐만 아니라, 사용자들이 서로 소통하고 함께 즐길 수 있는 경험을 목표로 개발되었습니다.

해당 애플리케이션은 Android 기반으로 제작되었으며,  
KBO 선수 조회, 경기 일정/스코어/문자중계, WebSocket 실시간 채팅,  
메타버스 공간 상호작용 등 다양한 기능을 제공합니다.


<본 프로젝트는 공학경진대회 출품작입니다.>

---

# 주요 기능

- 선수 조회 (KBO 리그 전체)
- 경기 날짜 및 스코어 조회
- 실시간 문자중계(텍스트 기반)
- WebSocket 실시간 채팅
- 메타버스 공간(이동/상호작용/말풍선 대화)
- JWT 로그인 / 회원가입
- 팀별/개인별 순위 조회
- 게시판(게시글 작성/조회)
- 샵 기능(상품 등록/조회/삭제/장바구니/가상 결제)

---

# 기술 스택

- Language : Kotlin, Java, Python  
- IDE / Tools : Android Studio, IntelliJ, VS Code  
- Real-time : WebSocket  
- Architecture : MVVM

# My Contribution

## 1. 샵 기능 단독 개발
- 서버가 없는 환경에서 상품/장바구니/가상 결제 단계까지 전체 흐름 구현
- 상품 등록·삭제·수량 변경·총액 계산 로직 설계
- 단일 액티비티 기준으로 자연스러운 결제 흐름을 직접 구성
- 로컬 상태를 기반으로 데이터 일관성 유지 구조 설계

## 2. WebSocket 기반 실시간 채팅 개발 (핵심 기능)
- 문자중계 화면과 연동되는 실시간 채팅 기능 구현
- WebSocket 연결·재연결·종료 등 상태 관리 로직 작성
- 경기 이벤트 메시지를 파싱하여 UI로 전달하는 구조 구성
- 메시지량 증가 시 UI 지연을 최소화하기 위한 경량 처리

## 3. 메타버스 공간 구현 (핵심 기능)
- 키 입력 기반 캐릭터 이동 좌표 계산 로직 구현
- 특정 위치 도달 시 화면 전환되는 포탈 구조 구성
- WebSocket을 통한 사용자 상태 실시간 동기화
- 사용자 간 말풍선 대화 기능 구현
- 메타버스 배경/오브젝트 이미지 직접 제작하여 UI 구성

## 4. 로그인 / 회원가입 및 인증 흐름
- JWT 기반 인증 기능 구현
- Retrofit + Interceptor로 토큰 처리 자동화
- 로그인 → 홈 화면으로 이어지는 UX 정리

## 5. 전체 UI/UX 공동 설계 및 화면 구성
- 전반적인 앱 UI 구조 확립: 화면 전환, 컬러 시스템, 메뉴 구조 통일
- 각 페이지 레이아웃 개선 및 사용자 흐름 정리

## 6. 일정 캘린더 기능 개발
- 야구 직관 기록(사진, 메모) 저장 기능 구현
- 날짜 기반 데이터 관리 설계

# 회고

- 서버가 없는 환경에서 클라이언트만으로 샵 기능을 구축하며 상태 관리 및 데이터 흐름을 설계하는 경험을 함 
- WebSocket 기반의 실시간 처리를 통해 비동기 이벤트 처리와 UI 업데이트 최적화에 대한 이해가 깊어짐 
- 메타버스 공간을 구현하며 좌표 기반 이동 계산과 실시간 동기화의 복잡성을 해결하는 과정에서 많은 기술적 성장을 경험함
